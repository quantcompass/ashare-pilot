# ashare-pilot

A股策略研究与信号系统 —— quantcompass 旗下项目。

> ⚠️ **风险提示**：本项目用于策略研究与学习，**不构成投资建议**。第一阶段**只做分析、信号与回测，不做真实下单**。任何实盘交易请自行承担风险，并先经过充分的回测与模拟盘验证。

## 它能做什么（第一阶段）

- 📈 拉取 A股日线行情（**多源容错**：stockapi 主源 + 腾讯备源 + 本地缓存）
- 📐 计算技术指标（当前：SMA 移动平均）
- 🎯 双均线金叉/死叉策略生成买卖信号
- 🔁 回测：用历史数据验证策略收益、回撤、胜率
- 🔍 选股：扫描自选股池，输出当日买入候选（取数失败会**显式告警**，不会误报）
- 🔔 信号提醒（当前控制台输出，预留钉钉/邮件）
- 🔌 下单接口预留（PaperBroker 模拟盘；真实券商 QMT 日后接入）

## 数据源与 token 配置

数据源是**多源自动容错**的组合（外层还有本地缓存）：

```
CachedSource( FallbackSource([ StockApiSource(主), TencentSource(备) ]) )
```

- **主源 stockapi**（[stockapi.com.cn](https://www.stockapi.com.cn/)）：需要 token，前复权日线。
- **备源 腾讯**（akshare `stock_zh_a_hist_tx`）：**免费、无需 token**，主源失败/超时时自动接管。
- **本地缓存**：历史日线落盘到 `data/cache/*.parquet`，复用免重复联网；**今天的数据每次都联网拉**（盘中会变）。

### 配置 stockapi token（可选）

主源 token 从**环境变量**读取，**不写进代码、不入库**：

```bash
export STOCKAPI_TOKEN="你的token"
```

> 💡 **不配 token 也能用**：没设 `STOCKAPI_TOKEN` 时，主源会失败并**自动回落到腾讯免费源**，功能不受影响。配了 token 则优先用 stockapi。

### 1. `backtest` — 回测单只股票

```bash
uv run ashare-pilot backtest --symbol 600519 --start 20240101 --end 20241201
```

实测真实输出（贵州茅台，2024 年，双均线 5/20，stockapi 主源）：

```text
标的 600519  区间 20240101~20241201  双均线(5,20)
  期末资产：78,074.63
  总收益率：-21.93%
  最大回撤：22.25%
  交易次数：8
  胜    率：12.50%
```

**参数：**

| 参数 | 必填 | 默认 | 说明 |
|---|---|---|---|
| `--symbol` | ✅ | — | 6 位股票代码，如 `600519` |
| `--start` | | 一年前 | 起始日 `YYYYMMDD` |
| `--end` | | 今天 | 结束日 `YYYYMMDD` |
| `--fast` | | 5 | 快线窗口 |
| `--slow` | | 20 | 慢线窗口 |

### 2. `signal` — 看单只股票最新信号

```bash
uv run ashare-pilot signal --symbol 600519
```

实测真实输出：

```text
=== 600519 最新信号（2024-11-29） ===
收盘价 1474.26
动作：⚪ 无信号 持有/观望
```

参数同 `backtest`。

### 3. `screen` — 扫描自选股池

```bash
uv run ashare-pilot screen
```

自选股池在 `src/ashare_pilot/config.py` 的 `watchlist` 中配置，默认 5 只：
`000001 / 600519 / 000858 / 601318 / 600036`。

- 有命中：逐行列出 `代码 现价`
- 无命中：显示「今日无金叉买入信号」
- **若全部取数失败**：显式告警「⚠️ 全部 N 只取数失败，结果不可信」，**不会**伪装成「无信号」

## 架构

```
datasource → indicators → strategy → backtest / screener → notify → execution(预留)
   取数         指标         信号        回测 / 选股          提醒        下单
```

- **数据源层**：多源容错 + 缓存，对上层暴露统一数据契约（OHLC 必备，volume/amount 可选）。新增数据源只需实现 `DataSource.fetch_daily` 协议。
- **核心设计**：策略是纯函数（价格 → 信号），无状态、不下单。回测、选股、未来的实盘共用同一套策略代码，避免"回测赚钱、实盘逻辑漂移"的经典坑。

各层职责见 [docs/design.md](docs/design.md)。

## 已知局限

### 回测模型为简化版（v1）
当前回测假设：信号当根**收盘价全仓成交**（轻微前视）、不考虑涨跌停 / 滑点 / 最小 100 股一手 / 分仓。数字偏乐观，详见 [docs/design.md](docs/design.md)。

### 各数据源复权基准略有差异
stockapi 与腾讯都返回前复权，但复权基准时点不同，同一标的同一天的绝对价格可能略有差异；不影响趋势与信号逻辑。
